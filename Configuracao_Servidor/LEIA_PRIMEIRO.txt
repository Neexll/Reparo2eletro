═══════════════════════════════════════════════════════════════════════════
  ❓ PERGUNTAS FREQUENTES
═══════════════════════════════════════════════════════════════════════════

  Q: Qual método devo escolher?
  A: Leia QUAL_METODO_ESCOLHER.txt
     TL;DR: Build Executável para a maioria dos casos

  Q: Preciso instalar Python no servidor?
  A: Não, se usar Build Executável
     Sim, se usar Instalação Manual

  Q: Quanto tempo demora?
  A: Build Executável: 10 min (build) + 2 min (deploy)
     Instalação Manual: 5-10 min

  Q: Funciona na internet?
  A: Não, apenas rede local (mesma WiFi/rede)

  Q: E se eu não souber nada de programação?
  A: Use Build Executável - é só copiar e executar!

  Q: Posso testar antes de colocar no servidor?
  A: Sim! Execute localmente primeiro


═══════════════════════════════════════════════════════════════════════════
  🚦 FLUXO RECOMENDADO
═══════════════════════════════════════════════════════════════════════════

  PASSO 1: Decisão (5 minutos)
  └─ Leia: QUAL_METODO_ESCOLHER.txt

  PASSO 2: Preparação (10 minutos)
  ├─ BUILD: Execute build_server.bat
  └─ MANUAL: Execute instalar_servidor.bat

  PASSO 3: Deploy (5 minutos)
  ├─ BUILD: Copie dist_servidor para servidor
  └─ MANUAL: Configure no próprio servidor

  PASSO 4: Configuração (5 minutos)
  ├─ Configure firewall
  ├─ Inicie o servidor
  └─ Anote a URL

  PASSO 5: Distribuição (10 minutos)
  ├─ Execute: criar_atalho_funcionario.bat
  └─ Distribua atalhos para funcionários

  TOTAL: ~35 minutos
